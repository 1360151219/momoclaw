# Daily Greeting Skill

向用户发送一条融合记忆、近况和新闻的个性化每日问候。

这个 Skill 的核心价值：让用户感觉 MomoClaw 是一个真正了解自己的朋友，而不是一个冷冰冰的播报机器人。问候内容应该像朋友发来的消息一样自然——有温度、有信息量、有个人化的关怀。

---

## 执行流程

### Step 1: 信息收集

依次读取以下文件，构建用户画像：

1. **读取 `memory/SOUL.md`** — 确认自己的人格和语气风格
2. **读取 `memory/USER.md`** — 获取用户偏好、兴趣标签（Interest Tags 区域）、近期关注事项
3. **读取近 3 天的记忆文件** — `memory/YYYY-MM-DD.md`（今天、昨天、前天），了解近期对话亮点、待办事项、用户提到的计划

从以上信息中提取：
- 用户的兴趣领域（技术方向、关注的公司/产品、生活爱好等）
- 近期的重要事项（即将到来的会议、截止日期、计划等）
- 未完成的待办或用户上次提到的话题
- 用户的语气偏好（如果有记录的话）

如果某个文件不存在，跳过即可——不要因缺少某个文件而中断整个流程。

### Step 2: 新闻抓取

根据 Step 1 中获取的用户兴趣进行抓取相关新闻。

**推荐新闻源（根据用户兴趣选择 2-3 个即可，不需要全部抓取）：**
- 技术类：Hacker News (https://news.ycombinator.com/best)、GitHub Trending (https://github.com/trending)
- 中文科技：36Kr (https://36kr.com/hot-list/catalog)、少数派 (https://sspai.com)
- AI 领域：AI News (https://buttondown.com/ainews/archive)
- 通用：可根据 USER.md 中的兴趣自由选择其他合适的信息源

**抓取规则：**
- 每个源最多取 5 条标题
- 根据用户兴趣筛选出最相关的 **2-3 条**
- 如果 article-fetcher 失败，降级使用 browser 工具
- 如果所有抓取都失败，跳过新闻部分，仅发送基于记忆的问候——绝不能因为新闻抓取失败就不发问候

### Step 2.5: 惊喜彩蛋（Surprise Me）

每天从以下彩蛋类型中 **随机挑选 1 个** 融入问候，让每天的问候都有一个小惊喜。不要每天用同一种，保持新鲜感。

**彩蛋菜单（每天随机选 1 个）：**

1. **历史上的今天** — 搜索今天日期发生过的一件有趣的事（科技/文化/冷知识都行），用一句话讲出来。越冷门越好，避免"爱因斯坦生日"这种人人都知道的。比如："历史上的今天，1991 年 Linux 0.01 内核首次发布，Linus 当时只是觉得写着玩玩"。

2. **今日兔子洞** — 推荐一个跟用户兴趣 *沾边但意想不到* 的链接或话题。关键在于"沾边但意想不到"——用户研究 Rust，你可以推荐一篇"用 Rust 写的音乐合成器"；用户喜欢咖啡，你可以推荐"用流体力学模拟咖啡拉花的论文"。附一句勾起好奇心的描述。

3. **回调彩蛋** — 翻记忆文件，找到用户之前说过的一句有趣的话、一个梗、一个夸张的抱怨，在今天的问候中巧妙引用。比如用户上周说"这个 bug 改了三天我要秃了"，今天可以说"头发还好吗？希望这周的代码对你温柔一点"。这种个人化的回调最能让人会心一笑。

4. **迷你挑战** — 给用户一个跟兴趣相关的 30 秒小挑战或小实验。技术类："今天试试不用 if-else 重写一段逻辑？"；生活类："今天的咖啡试试加一点点盐，据说能降低苦味"。轻松有趣，不要布置作业的感觉。

5. **数字里程碑** — 如果从记忆中能推算出某个有意义的数字（比如跟 MomoClaw 对话的天数、连续沟通的周数、完成的第 N 个项目），用轻松的方式庆祝一下。"今天是我们认识的第 100 天了，时间过得真快！"

6. **灵感碎片** — 不是烂大街的鸡汤名言，而是跟用户最近关注的话题 *直接相关* 的一句精彩引用。用户在研究分布式系统？引用 Leslie Lamport 的经典吐槽。用户在纠结技术选型？引用 Knuth 的"premature optimization"。要有信息增量，不是空洞的激励。

7. **代码小彩蛋** — 偶尔用代码的方式说一句话。比如用用户最常用的语言写一行有趣的注释或单行代码。`while (�.is_hard()) { coffee.drink(); keep_going(); }` 这种。简短，会心一笑就够了。

**选择策略：** 不是真随机——根据当天的信息做最合适的选择。如果今天有明显的里程碑，优先用"数字里程碑"；如果记忆中有好玩的回调素材，优先用"回调彩蛋"；如果都没有特别的，再从其他类型里选。

### Step 3: 内容生成

综合以上信息，生成一条问候消息。

**内容结构：**
1. **开头** — 个性化问候（结合日期/星期、用户近况，不要千篇一律的"早上好"）
2. **惊喜彩蛋** — Step 2.5 选出的那个彩蛋，自然地融入（不要生硬地标注"今日彩蛋："，让它成为问候的有机组成部分）
3. **中间** — 新闻/资讯精选（2-3 条，每条附一句个性化点评，不要简单转述标题）
4. **结尾** — 温暖收尾（如果知道用户近期有什么安排，自然地提到并鼓励）

**语气要求：**
- 像一个了解你的朋友在聊天，不像机器人在播报
- 参考 SOUL.md 中的人格设定来决定语气
- 自然、温暖、有趣，可以适当幽默
- 如果用户近期心情不太好（从记忆中判断），语气更加温柔体贴

**格式要求：**
- 总长度控制在 **200-500 字**
- 可以使用 emoji 但不要过多（3-5 个即可）
- 新闻部分用列表格式，清晰易读
- 使用 markdown 标题语法（#）

**示例（仅供参考风格，不要照搬）：**

示例 A（回调彩蛋 + 新闻）：
```
☀️ 周三早上好！话说你昨天那句"再写一个 bug 我就转行卖煎饼"笑死我了——煎饼摊位先帮你留着，但今天的代码说不定会对你温柔一点。

📰 今天值得关注的：
• Anthropic 刚发了 Claude 的新版 MCP 协议文档，工具调用的灵活度又上了一个台阶
• GitHub Trending 上有个用 Rust 写的终端音乐播放器，star 涨得很猛，你可能会喜欢

对了，你提到周五的技术分享还差几页 slides，加油！需要帮忙整理大纲随时说 💪
```

示例 B（历史上的今天 + 兔子洞）：
```
🌤 早！今天是 4 月 21 号，历史上的今天，1997 年第一台深海自主机器人完成了马里亚纳海沟的探测——比写 CSS 居中还难的事，人类也做到了。

🐇 给你挖了个有趣的兔子洞：有人用 Rust + WebAssembly 实现了一个浏览器里的流体力学模拟器，拿来模拟咖啡拉花效果，视觉上超治愈 → [链接]

📰 快讯：
• React 团队发了新的 RFC，Server Actions 的 API 要改了
• 36Kr 报道字节在内部推 AI 辅助代码审查，据说覆盖率已经到 80%

新的一天，写点让自己骄傲的代码 ✨
```

### Step 4: 输出

直接输出生成的问候消息文本。不要加任何 meta 说明（如"以下是为你生成的问候"），直接就是要发给用户的内容本身。这条消息会被 CronService 自动推送到用户的飞书/微信渠道。

---

## 注意事项

- **不要编造新闻**：如果抓取不到，就不写新闻部分，绝不虚构任何信息
- **不要重复**：如果记得昨天问候过的内容，今天要有所不同
- **尊重隐私**：不要在问候中提及用户可能不想被提醒的敏感话题
- **简洁优先**：宁可精简也不要冗长，用户想看的是一条有温度的消息，不是一篇文章
- **节日彩蛋**：如果今天是节假日或用户生日（从 USER.md 获取），在问候中自然融入节日元素
